# 高校软著登记申请表填写

## 目标

根据用户提供的软件项目信息，产出《计算机软件著作权登记申请表》的可填写草稿，并指出缺失项、冲突项和不符合验收标准的字段。只处理登记申请表本身；软件说明书、代码鉴别材料和校内附件表由其他 Skill 处理。

## 输入判断

先确认用户要填写的是“计算机软件著作权登记申请表”。如果用户要求“软件说明书”“源代码文件”“附件1申请信息表”或“整套材料打包”，说明当前 Skill 只覆盖申请表，并建议拆分处理。

读取详细规则时使用：

- `references/application-form-source.md`：用户粘贴的原始表单结构存档。
- `references/field-rules.md`：字段填写规则、默认值、必问项和验收标准。

## 工作流

### Step 1: 收集关键信息

先整理用户已提供的信息，再列出缺失项。不要编造以下字段：

- 软件全称、简称、版本号、开发完成日期
- 开发/运行硬件环境、操作系统、开发工具、运行支撑环境
- 源程序量、开发目的、面向领域/行业、主要功能、技术特点说明
- 著作权人名称、类别、证件类型、证件号码
- 联系人、电话、邮箱、手机号

若缺失项较多，先输出“待补充信息清单”，不要直接生成完整表。

### Step 2: 应用默认填写规则

在用户未另行说明时，使用本项目当前默认规则：

| 字段 | 默认值 |
| --- | --- |
| 软件分类 | 应用软件 |
| 软件作品说明 | 原创 |
| 发表状态 | 未发表 |
| 开发方式 | 独立开发 |
| 权利取得方式 | 原始取得 |
| 权利范围 | 全部 |
| 编程语言 | Python |
| 软件涉及技术特点 | 信息安全软件、人工智能软件、智慧城市软件 |
| 申请方式 | 由著作权人申请 |
| 申请人详细地址 | 按申请人或著作权人实际地址填写 |
| 邮政编码 | 按申请人或著作权人实际地址填写 |
| 软件鉴别材料 | 多种文档：平台原型、说明文档 |

默认值可以被用户明确覆盖。覆盖后检查是否违反表单限制，例如软件技术特点最多选三项。

### Step 3: 生成申请表草稿

按原表单顺序输出，不要重新发明字段名。推荐用 Markdown 表格输出“字段 / 填写内容 / 状态或备注”。

字段状态使用：

- `已填`：有明确值，且满足规则。
- `待补充`：缺必要信息。
- `需确认`：有值但可能不合理、冲突或不符合常见填法。
- `不适用`：在当前默认路径下不用填写，例如未发表时的首次发表信息。

### Step 4: 校验验收标准

生成草稿后逐项检查：

- 软件分类四选一。
- 软件作品说明若选择“修改”，修改说明不超过 100 字，并说明新增模块、程序行号、翻译文本或原登记号。
- 开发完成日期必须完整到年月日。
- 未发表时，首次发表日期和地点应为空或标注不适用。
- 开发方式、权利取得方式、权利范围不可互相冲突。
- 编程语言最多 10 项。
- 源程序量必须是正整数，单位为“行”。
- 开发目的不超过 50 字。
- 面向领域/行业不超过 50 字。
- 软件主要功能不超过 200 字。
- 软件涉及技术特点最多选择 3 项。
- 申请人地址、邮编、联系人、电话、邮箱、手机号完整。
- 鉴别材料说明包含源程序与文档页眉、页码、行数要求。

### Step 5: 输出缺口和下一步

最后输出：

1. 可直接填写进申请表的草稿。
2. 待用户补充的信息清单。
3. 验收风险清单。
4. 可交给软件说明书 Skill 复用的“项目事实摘要”。
5. 下一步建议，例如继续生成软件说明书或源代码鉴别材料。

项目事实摘要应只包含用户已确认或表单中已明确填写的信息，重点整理：软件全称、版本号、开发完成日期、软件用途、目标用户、主要功能、技术特点、运行环境、开发工具、编程语言、源程序量、著作权人和鉴别材料类型。不要把待补充项伪装成事实。

## 输出格式

使用以下结构：

```markdown
## 申请表填写草稿

| 模块 | 字段 | 填写内容 | 状态/备注 |
| --- | --- | --- | --- |

## 待补充信息

- ...

## 验收检查

- [ ] ...

## 风险提示

- ...

## 给软件说明书 Skill 的项目事实摘要

| 项目 | 内容 |
| --- | --- |
```

## 重要约束

- 不要声称已经生成或修改 `.doc` 文件，除非实际完成了文件写入。
- 不要把占位符当成真实信息。例如原表里“证件号码 4”应标记为异常或待确认。
- 不要自动选择“已发表”，除非用户提供首次发表日期和地点。
- 对学校、学院、团队、学生个人、项目负责人等权利归属信息保持谨慎，必须由用户确认。
- 不提供法律结论；只根据表单填写规则做材料整理和一致性校验。
