# 计算机软件著作权登记申请表原始结构存档

本文件存档用户粘贴的《计算机软件著作权登记申请表》字段结构和填写提示，用于后续 Skill 编写与校验。它不是最终填写结果。

## 软件基本信息

| 字段 | 原始要求或选项 |
| --- | --- |
| 软件全称 | 待填写 |
| 软件简称 | 待填写 |
| 软件分类 | 应用软件、嵌入式软件、中间件、操作系统，四选其一；当前样例勾选“应用软件” |
| 版本号 | 待填写 |
| 软件作品说明 | 原创；或修改，含翻译软件、合成软件 |

修改软件填写说明：申请软件属于在原有软件基础上形成的修改软件的，应填写修改软件作品说明，简短明确阐述“该软件新增功能模块”或“程序的第10～100行”、“全部翻译文本”等。对翻译或合成软件应对其取得著作权的材料及所编入的材料进行概括性描述。说明字数限于100字之内。若原有软件已经进行登记的，应注明原软件著作权登记号。

| 字段 | 原始要求或选项 |
| --- | --- |
| 开发完成日期 | 年、月、日 |
| 发表状态 | 已发表、未发表；当前样例勾选“未发表” |
| 已发表信息 | 首次发表日期；首次发表地点，含国家、城市 |
| 开发方式 | 独立开发、合作开发、委托开发、下达任务开发；当前样例勾选“独立开发” |

合作开发说明：指两人或两人以上依据合作协议共同开发完成的软件，需要提供申请人享有权利的合作开发协议。

## 软件鉴别材料

| 字段 | 原始要求或选项 |
| --- | --- |
| 一般交存 | 提交源程序前连续的30页和后连续的30页；提交任何一种文档的前连续的30页和后连续的30页 |
| 一种文档 | 可选 |
| 多种文档 | 当前样例勾选“多种文档”，种类为：平台原型、说明文档 |
| 例外交存 | 使用黑色宽斜线覆盖；前10页和任选连续的50页；目标程序连续前、后各30页和源程序任选连续的20页 |

软件鉴别材料说明：源程序和文档应在页眉上标注相应的软件名称和版本号，右上角应标注页码。源程序每页不少于50行，文档每页不少于30行。
