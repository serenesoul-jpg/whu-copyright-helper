# 软件说明书推荐目录

目标：生成内容充实、图文并茂、可作为软著文档鉴别材料的一份说明书。默认正文不少于 20000 个中文字符，复制到 Word 后按 20 页以上规划；页数分配只是内容体量参考，具体排版由后续 Word 处理完成。

完整长文档的字数预算、分卷规则和长度自检见 `long-form-output-protocol.md`。

## 推荐结构

| 章节 | 内容 | 建议内容体量 |
| --- | --- | --- |
| 封面 | 软件全称、版本号、著作权人、日期 | 1 |
| 目录 | 自动目录或手工目录；从目录开始编第1页时按实际要求处理 | 1-2 |
| 1. 引言 | 编写目的、适用对象、术语、参考资料 | 1800字以上，约3-5页 |
| 2. 软件概述 | 软件背景、建设目标、应用场景、用户角色 | 2600字以上，约5-8页 |
| 3. 运行环境 | 硬件环境、操作系统、浏览器/依赖、部署要求 | 2200字以上，约3-5页 |
| 4. 系统总体设计 | 总体架构、模块组成、数据流、AI/算法架构 | 3200字以上，约6-10页 |
| 5. 功能模块说明 | 按核心模块逐章说明用途、流程、截图、输入输出 | 8000字以上，约25-35页 |
| 6. 操作流程 | 登录、配置、数据录入、分析、查询、导出等完整流程 | 3000字以上，约8-12页 |
| 7. 测试与异常处理 | 功能测试、边界场景、错误提示、权限控制 | 2400字以上，约4-6页 |
| 8. 维护与附录 | 常见问题、术语、版本说明、图表索引 | 1800字以上，约3-5页 |

## 每个功能模块的写法

```text
5.x 模块名称
5.x.1 模块概述
5.x.2 入口与权限
5.x.3 操作步骤
5.x.4 页面截图与字段说明
5.x.5 输入输出数据
5.x.6 异常提示与处理
```

## 内容展开原则

- 每张截图都要补充图题、图注、操作步骤和字段说明。
- 章节标题、图题、表题要连续编号。
- 同一模块的截图顺序应与用户实际操作顺序一致。
- 优先用真实功能、真实流程和真实数据解释扩展内容，不用重复套话撑篇幅。
- 完整说明书请求下，不能只输出目录或前两章样例；必须继续展开正文直到达到长度目标或说明缺少哪些真实材料导致无法达标。
