# 截图、架构图和图表组织规则

## 截图清单

优先准备这些截图：

| 类型 | 示例 |
| --- | --- |
| 启动/入口 | 登录页、首页、项目入口、命令行启动结果 |
| 核心流程 | 数据录入、任务创建、分析执行、结果生成 |
| 结果展示 | 报告页、图表页、风险评分、导出结果 |
| 管理功能 | 用户管理、权限配置、参数设置、日志管理 |
| 异常场景 | 输入校验失败、权限不足、任务失败提示 |
| 输出材料 | 导出的文档、报表、文件包、分析结果 |

## 图表清单

建议加入：

- 系统总体架构图。
- AI/算法处理流程图。
- 数据流图。
- 用户操作流程图。
- 模块关系图。
- 测试流程或验收流程图。

## 图片说明规则

每张图必须配套：

```text
图 5-1 登录页面
该页面用于系统用户输入账号和密码进入平台。系统对用户名、密码格式和权限状态进行校验，校验通过后进入首页，校验失败时展示错误提示。
```

## AI 架构图要求

AI 架构图应说明真实处理链路，例如：

```text
用户输入/项目数据 -> 数据预处理 -> 特征提取 -> 模型推理/规则判断 -> 结果解释 -> 报告生成
```

如果项目只是普通 CRUD 或工具系统，不要强行写 AI 架构图。可以改用系统架构图和业务流程图。
