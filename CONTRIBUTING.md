# Contributing

欢迎改进 University Software Copyright Helper。这个项目的核心目标是让 Codex Skills 更稳定地辅助准备高校软件著作权登记材料。

## 贡献方向

- 改进申请表字段规则、默认值和冲突检查。
- 改进软件说明书的章节模板、长文档输出协议和验收清单。
- 增加真实但脱敏的示例提示词和评估用例。
- 补充源代码鉴别材料、校内附件表或 `.docx` 生成脚本。
- 修正文档中的过时要求、表述不清或误触发问题。

## 修改 Skill 的原则

- 保持一个 Skill 只解决一类明确任务。
- `SKILL.md` 保持精简，核心流程放这里，详细规则放 `references/`。
- `SKILL.md` 的 YAML frontmatter 只保留 `name` 和 `description`。
- `description` 必须同时说明适用场景和不适用场景。
- 不要把学校、个人、项目、证件、联系方式等真实敏感信息提交到仓库。
- 不要承诺“包过”“一定通过”，也不要给法律结论。

## 提交流程

1. Fork 本仓库并创建分支。
2. 修改相关 Skill、reference 或文档。
3. 用真实或接近真实的脱敏输入试跑一遍。
4. 检查输出是否包含缺失项、风险提示和验收自检。
5. 提交 Pull Request，并说明修改目的、影响范围和测试方式。

## 推荐自检

提交前请确认：

- 新增规则能被 `SKILL.md` 直接发现和引用。
- 没有提交真实申请表、说明书、截图、证件号、手机号、邮箱或未公开项目材料。
- 说明书相关修改没有削弱 20000 中文字符以上长文档目标。
- 申请表相关修改没有让模型自动编造日期、权利人、证件号、联系人和代码行数。
- Markdown 链接和表格可以正常阅读。
